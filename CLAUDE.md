# tailscaled-rs

An independent, from-scratch Rust **system daemon** that joins a WireGuard-based mesh overlay by
speaking the Tailscale control protocol. It is the long-running, IPC-controlled *daemon* layer
(`tailnetd` + the `tnet` CLI) built on top of the embeddable
[`tailscale-rs`](https://github.com/GeiserX/tailscale-rs) engine.

## Architecture

- **Engine** (`tailscale-rs`, a dependency): cryptography + data plane — control Noise handshake,
  network-map client, magicsock (UDP + disco), DERP, WireGuard, smoltcp netstack, packet filter.
- **Daemon** (this repo): lifecycle + intent — the IPN state machine (`src/ipn.rs`), persisted
  Prefs (`src/prefs.rs`), the LocalAPI server over a Unix socket (`src/server.rs`), and the wire
  DTOs (`src/localapi.rs`). Binaries: `tailnetd` (daemon) and `tnet` (thin CLI).

The daemon owns the lifecycle and intent; it never touches packets — the engine does.

## Conventions

- Rust edition 2024. The engine is pinned by `rev` in `Cargo.toml`; `Cargo.lock` is committed.
- The engine requires `TS_RS_EXPERIMENT=this_is_unstable_software` at runtime.
- CI gates: `cargo fmt --check`, `cargo clippy -- -D warnings`, `cargo test`, release build. Run
  them locally before pushing.
- Conventional commits. BSD-3-Clause; retain the upstream Tailscale copyright on derived portions.
- This is an unofficial project: not affiliated with Tailscale Inc. or the WireGuard project; use
  those names only nominatively.

## Status

Experimental MVP (Phase 1): joins a real tailnet with a pre-auth key in userspace-networking mode,
state machine + persisted prefs + LocalAPI `status`/`up`/`down`. See `docs/DESIGN.md` for the
phased plan toward a full daemon.
